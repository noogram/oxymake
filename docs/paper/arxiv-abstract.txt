Research workflows are executed repeatedly as code, data, and machines change. Each run must decide which results are still valid and which must be recomputed. This decision commonly derives from the state of one working copy, so it does not survive a new checkout, a new machine, or a new collaborator. We present OxyMake, a workflow engine that derives the decision from the declared content of the workflow itself: re-running converges to the declared outputs, performs only the work whose inputs changed, and gives the same answer in any copy of the project. The same workflow runs on a laptop or on a cluster, driven by a person or by a program. We describe the design, compare scheduling cost and repeated execution with an established engine, and analyze the safety of concurrent runs. The guarantees cover the inputs a workflow declares.
